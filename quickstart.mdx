---
title: "Quickstart: clone and run Tic Tac Toe in your browser"
description: "Clone the repository, open index.html in any modern browser, and start playing in under a minute — no install or build step needed."
---

Tic Tac Toe Game has no dependencies and no build process, so there are multiple ways to get it running. You can open the HTML file directly from your filesystem or serve it through a local development server — both approaches work equally well.

<Steps>
  <Step title="Clone the repository">
    Download the project to your machine using Git:

    ```bash
    git clone https://github.com/samuelitoo01/Tic-Tac-Toe-Game.git && cd Tic-Tac-Toe-Game
    ```
  </Step>

  <Step title="Open in a browser">
    Choose how you want to run the game:

    <Tabs>
      <Tab title="Direct open">
        Double-click `index.html` in your file manager, or drag it into an open browser window. The game loads immediately — no server needed.
      </Tab>
      <Tab title="Local server">
        Serving through a local server avoids any browser restrictions on local file access. Pick one of the options below:

        ```bash
        # Python 3
        python -m http.server 8000
        ```

        ```bash
        # Node.js (no install required)
        npx http-server
        ```

        Then open `http://localhost:8000` in your browser.
      </Tab>
    </Tabs>
  </Step>

  <Step title="Play the game">
    Once the page loads, you will see the 3×3 grid and a "Jugador Actual" (current player) indicator showing whose turn it is. Player X always goes first. Click any empty cell to place your marker. The game switches turns automatically, detects wins and draws, and lets you restart at any time with the "Reiniciar juego" button.
  </Step>
</Steps>

<Note>
  No internet connection is required after cloning. All game assets — HTML, CSS, and JavaScript — are local files that run entirely in the browser.
</Note>

## Project files

| File | Description |
|------|-------------|
| `index.html` | The main HTML document. Defines the page structure: the game title, current-player display, the nine cell elements that make up the board, the restart button, and the footer. Also loads `styles.css` and `game.js`. |
| `game.js` | Contains all game logic and UI interaction. Manages the 3×3 board matrix, tracks the active player and game state, handles click events on each cell, and checks for wins across rows, columns, and diagonals as well as draw conditions. |
| `styles.css` | Provides all visual styling. Uses CSS Grid to lay out the 3×3 board, centers the game on the page, and styles the cells, title, restart button, and footer. |
